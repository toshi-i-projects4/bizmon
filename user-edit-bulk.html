CSV一括招待・更新・取消

CSVを読み込み、内容を確認してから Firestore の pendingUsers を一括処理します。Firebase Authentication のユーザー作成は行いません。従業員本人が employee-signup.html で初回登録すると users に本登録されます。

CSV列：action, employeeCode, name, email, department, position, role, status, isApplicant, note action：add / update / cancel role：admin / user status：pending / cancelled 重要：uid列は不要です。 companyId / companyName はCSVに入れません。ログイン中管理者の会社情報を自動付与します。 registered の pendingUsers は本人登録済みのため、この画面では更新・取消できません。
注意：この画面は pendingUsers の一括登録用です。登録後、管理者は表示された初回登録URLを従業員へ案内してください。招待メールの完全自動送信は無料プランのみでは行いません。
