CSV一括登録・更新・無効化

CSVを読み込み、内容を確認してから Firestore の users を一括処理します。完全削除は行わず、disable により無効化します。

CSV列：action, uid, employeeCode, name, email, department, position, role, status, isApplicant, note action：add / update / disable role：admin / user status：active / inactive 重要：add の場合は、先に Firebase Authentication にユーザーを作成し、その UID を uid 列に入力してください。 companyId / companyName はCSVに入れません。ログイン中管理者の会社情報を自動付与します。
注意：この画面では Firebase Authentication のユーザー作成は行いません。新規追加（add）するユーザーは、事前に Authentication 側で作成済みである必要があります。

取込内容の確認
