管理対象の切替

pendingUsers方式に合わせ、招待中ユーザーと本登録済みユーザーを分けて管理します。新規招待は user-register.html、複数名の招待は user-edit-bulk.html を利用してください。

・招待中ユーザー：pendingUsers の内容を個別に更新・取消できます。 ・本登録済みユーザー：users の部署・役職・権限・状態などを個別に更新できます。 ・この画面では Firebase Authentication の新規作成や削除は行いません。

招待中・登録済み予定者の検索

自社の pendingUsers を検索し、選択して編集します。registered のユーザーは本人登録済みのため、この画面では更新・取消できません。
